Task: Implement i686 64-bit division runtime helpers
Status: in_progress

Problem:
On i686 (32-bit x86), 64-bit integer division/modulo generates calls to
__divdi3, __udivdi3, __moddi3, __umoddi3 (GCC runtime library functions).
When building standalone C libraries like musl that don't link with libgcc,
these symbols are missing, causing link failures.

Example: musl's vfprintf.o references __udivdi3/__umoddi3 for %lld formatting.
This causes musl math test failure on i686 (big=3567587328 instead of 1000000000000).

Fix:
- Auto-emit weak implementations of __divdi3, __udivdi3, __moddi3, __umoddi3
  in the i686 linker/codegen output when these symbols are referenced but undefined
- These implement 64-bit division using 32-bit div instructions
- Use .weak linkage so user-provided implementations take precedence
