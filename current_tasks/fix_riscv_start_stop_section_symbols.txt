Fix RISC-V linker __start_/__stop_ section symbols

Problem: The RISC-V linker emit_exec.rs uses `sec.vaddr` to resolve __start_X and
__stop_X symbols, but the MergedSection.vaddr field is never updated (stays 0).
The actual section addresses are in the `section_vaddrs[]` array.

This causes a SIGSEGV in programs that use the GNU ld __start_/__stop_ feature
(e.g. the start-stop-section-symbols test case).

Fix: Use section_vaddrs[index] instead of sec.vaddr when resolving these symbols.
