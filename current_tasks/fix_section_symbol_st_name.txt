Fix: set st_name=0 for STT_SECTION symbols in ELF .o output

CCC currently emits section symbols (STT_SECTION) with st_name pointing to
the section name string in .strtab. GCC emits section symbols with st_name=0.

This causes the Linux kernel's modpost tool to fail with:
  "ERROR: modpost: .export_symbol section contains strange symbol '.export_symbol'"

modpost's find_fromsym matches the section symbol named ".export_symbol" when
searching for __export_symbol_* labels, because ".export_symbol" passes is_valid_name().

Fix: set st_name=0 for all STT_SECTION symbols in write_relocatable_object().
