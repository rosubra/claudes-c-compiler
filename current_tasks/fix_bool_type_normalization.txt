Fix _Bool type normalization for pointer stores

The problem: Storing through a _Bool pointer (*pval = 100) does not normalize the value
to 0 or 1. Direct _Bool variable assignment works correctly because is_bool_lvalue()
only handles Expr::Identifier, not Expr::Deref.

Plan:
1. Add CType::Bool variant to the type system (currently _Bool maps to UChar)
2. Update is_bool_lvalue to handle Deref, ArraySubscript, and MemberAccess expressions
   by checking the CType of the pointed-to/element type
3. Ensure all stores to _Bool-typed lvalues normalize the value
4. Add regression test
