Task: Fix complex array member initialization in structs

Bug: When initializing a struct containing a _Complex array field, integer
initializers produce zeros instead of proper complex values.

Example:
  struct S { _Complex double a[2]; };
  struct S s = { 1, 2 };
  // s.a[0] == 0.0+0.0i (BUG - should be 1.0+0.0i)
  // s.a[1] == 0.0+0.0i (BUG - should be 2.0+0.0i)

Root cause: emit_struct_init in stmt.rs handles the CType::Array arm without
checking if the element type is complex. It uses emit_init_expr_to_offset_bool
which does IrType::from_ctype on complex types, getting IrType::Ptr, and then
does implicit cast to pointer rather than constructing a {real, 0.0} pair.

Fix locations:
1. stmt.rs: emit_struct_init - CType::Array arm needs complex element handling
2. global_init_bytes.rs: fill_array_field needs complex element handling
3. global_init_compound.rs: emit_compound_field_init needs array-of-complex check

Affects all three backends (x86, ARM, RISC-V) since the bug is in IR lowering.
