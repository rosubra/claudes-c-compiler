Fix macro parameter substitution prefix-matching bug

Bug: When a .macro has parameters like `orig` and `orig_len`, the naive
String::replace in declaration order causes `\orig` to match inside
`\orig_len`, producing corrupted output like `140b_len` instead of
the value of `orig_len`.

This causes undefined references to `140b_len` and `143f_len` in the
Linux kernel's .altinstructions section when building in standalone mode.

Fix: Sort macro parameters by name length (longest first) before
substitution, matching what the ARM parser already does correctly.

Files to fix:
- src/backend/asm_preprocess.rs (expand_macros, expand_macros_with)
- src/backend/x86/assembler/parser.rs (expand_gas_macros_with_state)
