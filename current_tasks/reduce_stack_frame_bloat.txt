Reduce stack frame bloat by skipping stack slot allocation for register-assigned values

Problem: The compiler allocates stack slots for ALL IR values in calculate_stack_space_common(),
then runs register allocation which assigns some values to callee-saved registers. But the
stack slots for register-assigned values are never reclaimed, wasting significant stack space.

For a simple recursive function like:
  int recurse(int n) { if (n <= 0) return 0; return recurse(n-1)+1; }
GCC allocates 16 bytes, our compiler allocates 160 bytes (10x larger).

This directly causes the postgres plpgsql recursive test to fail because PostgreSQL's
max_stack_depth check triggers prematurely.

Fix: Run register allocation BEFORE stack space computation, and pass the register
assignment map to calculate_stack_space_common() so it can skip allocating stack slots
for values that have been assigned to registers.

Files to modify:
- src/backend/generation.rs (calculate_stack_space_common to accept reg_assignments)
- src/backend/x86/codegen/codegen.rs (reorder regalloc before stack computation)
- src/backend/arm/codegen/codegen.rs (same)
- src/backend/riscv/codegen/codegen.rs (same)
