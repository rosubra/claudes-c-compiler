Task: Fix _Alignof to respect _Alignas on variables
Status: in_progress

Problem: _Alignof(x) where x is declared with _Alignas(32) returns the natural
type alignment (e.g., 4 for int) instead of the declared alignment (32).
This is a C11 standards violation (6.2.8p3).

Root cause: _Alignas alignment overrides on variables are used when allocating
stack space (alloca) and global data, but are never stored in VarInfo or
LocalInfo. When _Alignof evaluates on a variable identifier, it falls through
to ctype_align() which returns the natural type alignment.

Fix:
- Add explicit_alignment: Option<usize> to VarInfo
- Store the _Alignas alignment when creating LocalInfo/GlobalInfo
- In alignof_expr(), when the expression is an identifier, check for
  explicit_alignment and return it if present
