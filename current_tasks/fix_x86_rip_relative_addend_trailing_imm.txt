Fix x86 assembler: RIP-relative relocation addend off-by-N for instructions with trailing immediates.

Bug: encode_modrm_mem always uses addend -4 for R_X86_64_PC32 relocations, but
instructions like `addl $1, sym(%rip)` have an imm8 byte after the disp32 field,
requiring addend -5. Similarly, `movl $imm32, sym(%rip)` needs addend -8.

The fix adds a trailing_bytes parameter to encode_modrm_mem (or post-adjusts the
relocation) so the addend correctly accounts for all instruction bytes after disp32.

Affected test cases: asm-mem-operand-symbol-x86 (and likely asm-P-global-addr-offset-x86)
Started: 2026-02-05
