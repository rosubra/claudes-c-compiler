Refactor IR emission patterns in expr.rs using helper methods

Goal: Reduce the massive expr.rs (3600+ lines) by adding emit_binop/emit_cmp/emit_cast
helper methods and using them to dramatically shrink repetitive IR emission patterns.

Changes:
1. Add helper methods to Lowerer:
   - emit_binop(op, lhs, rhs, ty) -> Value  (replaces 4-line fresh_value+BinOp pattern)
   - emit_cmp(op, lhs, rhs, ty) -> Value    (replaces 4-line fresh_value+Cmp pattern)
   - emit_cast_val(src, from_ty, to_ty) -> Value (replaces 3-line fresh_value+Cast pattern)

2. Refactor lower_builtin_fpclassify (~210 lines -> ~80 lines) using the helpers

3. Consolidate FP classification builtins: isnan/isinf/isfinite/isnormal/signbit/isinf_sign
   all share the same setup pattern (lower_fp_classify_arg, bitcast_float_to_int, fp_masks)
   and could be a single parameterized function dispatching on classification type.

4. Apply helpers across the rest of expr.rs where the 4-line pattern repeats.

Expected net reduction: ~400-600 lines from expr.rs.

Status: in progress
