Task: Fix sizeof for pointer arrays initialized with string literals

Bug: When a pointer array (e.g., `const char *arr[]`) is initialized with string
literal expressions (e.g., `{"hello"}`), the compiler incorrectly computes the
array size by counting the characters in the string literal instead of counting
the number of initializer expressions.

Example:
  static const char *a[] = { "hello" };
  sizeof(a) returns 48 (6 chars * 8 byte pointer) instead of 8 (1 pointer)

Root cause: compute_init_list_array_size_for_char_array in const_eval_init_size.rs
treats string literals as char-array initialization even when the array element type
is a pointer (not a bare char).

Fix: In global_decl.rs fixup_unsized_array, when is_array_of_pointers is true,
use compute_init_list_array_size instead of the char-array variant.

Impact: Fixes flatcc segfault and potentially other projects.
