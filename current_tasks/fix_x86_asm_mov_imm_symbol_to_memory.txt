Fix x86 assembler: movq/movl $symbol, memory(%rip) encoding

The x86 assembler's encode_mov_imm_mem function only handles
ImmediateValue::Integer. It needs to also handle Symbol and
SymbolPlusOffset by emitting R_X86_64_32S (for movq, sign-extended)
or R_X86_64_32 (for movl) relocations.

This breaks the x86 defconfig kernel build on wakeup_64.S which uses:
    movq $.Lresume_point, saved_rip(%rip)

Fix: Add Symbol/SymbolPlusOffset handling to encode_mov_imm_mem in
src/backend/x86/assembler/encoder/gp_integer.rs
