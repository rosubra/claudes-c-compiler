Task: Fix static function linkage (emitting .globl for static functions)

Problem:
- `static` keyword on function definitions is parsed but silently dropped
- FunctionDef AST node has no `is_static` field
- Sema always assigns StorageClass::Extern to functions
- IrFunction has no `is_static` field
- Codegen unconditionally emits `.globl` for all functions
- This causes duplicate symbol errors when the same static function name exists
  in multiple translation units (blocking Lua build)

Fix:
1. Add `is_static` field to FunctionDef AST node
2. Propagate parsing_static flag to FunctionDef
3. Pass is_static through sema to IR lowering
4. Add `is_static` field to IrFunction
5. In codegen, only emit `.globl` when !is_static
6. Also handle `inline` functions without `static` (emit as local by default)

Status: In progress
