Fix asm goto label substitution for kernel compilation

Problem:
- The compiler parses `asm goto(...)` syntax and recognizes the goto qualifier
- The fourth colon section (goto labels) is consumed but discarded (not stored in AST)
- When inline assembly references labels via %l[name] or %l0, the substitution does NOT
  replace these with the actual label names
- This causes assembly failures like: bad register name '%l[l_yes]'
- Many kernel files use asm goto (via static_branch, alternatives, etc.) and fail to compile

Fix:
1. Store goto labels in the AST's InlineAsm node
2. Pass goto labels through IR lowering to the IR InlineAsm instruction
3. Pass goto labels through to backend emission
4. In template substitution, replace %l[name] and %lN with actual label names
5. Handle the interaction with existing %l (32-bit register) modifier on x86

Files affected:
- frontend/parser/ast.rs (add goto_labels field to InlineAsm)
- frontend/parser/statements.rs (parse and store goto labels)
- ir/ir.rs (add goto_labels to IR InlineAsm)
- ir/lowering/stmt.rs (lower goto labels)
- backend/inline_asm.rs (pass labels through)
- backend/x86/codegen/inline_asm.rs (substitute %l[name])
- backend/traits.rs (update trait signature)
- backend/generation.rs (pass labels through)
- backend/arm/codegen/inline_asm.rs (substitute labels)
- backend/riscv/codegen/inline_asm.rs (substitute labels)

Status: in progress
