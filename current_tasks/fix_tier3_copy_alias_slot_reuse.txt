Task: Fix Tier 3 copy-alias slot reuse causing SIGSEGV in libsodium x86
Status: in_progress
Branch: fix_tier3_copy_alias_slot_reuse

Problem:
  libsodium secretbox and box tests crash with SIGSEGV when compiled with ccc-x86.
  Root cause: Tier 3 block-local greedy slot reuse does not extend a copy-alias root's
  last_use to cover the alias's uses. When a copy-aliased value is used as an Intrinsic
  dest_ptr, the root's slot is freed too early and reused by a later I128->I64 Cast
  result, overwriting the destination pointer with zero.

Fix:
  In assign_tier3_block_local_slots, when computing last_use for block-local values,
  extend the copy-alias root's last_use whenever a copy-aliased value is used as an
  operand or value reference in any instruction.

Files changed:
  - src/backend/stack_layout.rs: Extend copy-alias root liveness in Tier 3 analysis
  - tests/sse-storeu-copy-alias-slot-reuse/: Regression test
