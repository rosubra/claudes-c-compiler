Task: Fix i686 i128 store register clobber bug

Problem: When storing a 128-bit (I128/U128) value via an indirect pointer on i686,
the emit_store_default function in traits.rs clobbers callee-saved registers
(esi/edi) used for the accumulator pair save, which destroys the destination
pointer if it was register-allocated to esi or edi.

Sequence that fails:
1. emit_load_acc_pair(val) -- loads 64-bit value into eax:edx
2. emit_save_acc_pair()   -- mov %eax, %esi; mov %edx, %edi (CLOBBERS pointer!)
3. emit_load_ptr_from_slot() -- loads pointer from esi/edi into ecx (WRONG VALUE!)
4. emit_store_pair_indirect() -- writes to wrong address -> SIGSEGV

Manifests as: libsodium crypto_sign SIGSEGV in fe25519_reduce (Ed25519 operations)
Root cause: t[2] = f[2] where t is uint128_t array, pointer allocated to esi

Fix: Reorder operations so pointer is loaded into ecx BEFORE esi/edi are clobbered.
