Fix -Wreturn-type remaining false positives (dead break, noreturn preservation)

Incremental fixes on top of f251ff09 (comprehensive -Wreturn-type fix):

1. Dead break after return: segment_outcome() and stmt_switch_outcome() for
   Compound now walk items sequentially. If an item diverges (return/goto/etc.),
   subsequent items are dead code and don't affect the outcome. This fixes
   false positives for `return NULL; break;` patterns (argtable3 regex engine).

2. Noreturn preservation: function definitions now inherit is_noreturn from
   prior prototype declarations. Previously, `__attribute__((noreturn)) void f();
   void f() { abort(); }` would overwrite is_noreturn=true with false.

3. Additional noreturn builtins: quick_exit, __assert_fail, __assert_rtn,
   longjmp, siglongjmp, __longjmp_chk now recognized as noreturn.

4. Cast handling: is_noreturn_call() now handles Expr::Cast wrapping a
   noreturn function call (e.g., (void)abort()).
