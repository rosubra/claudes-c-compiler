Task: Add .gnu.version and .gnu.version_r output to x86 linker

Problem: The x86 linker does not emit GNU symbol versioning sections
(.gnu.version / .gnu.version_r) in dynamically-linked executables. When a
symbol like `realpath` has multiple versions in glibc (realpath@GLIBC_2.2.5
and realpath@@GLIBC_2.3), the dynamic linker resolves to the oldest/base
version instead of the default (@@) version. This causes Janet's test suite
to fail because realpath(path, NULL) returns EINVAL in the old version.

Fix: When emitting a dynamic executable, look up the default version for
each imported symbol from the shared library's .gnu.version/.gnu.versym
data, and emit .gnu.version and .gnu.version_r sections in the output.
